# 从零制作一个章节汉化补丁的流程

## 最高原则

- 只以 JP 原文和游戏实际读取槽为主要依据。
- 不使用英文槽作为翻译依据。
- 不使用英文兜底、旧中文兜底、模糊匹配回写。
- JP 为空不等于游戏不显示，必须对照 egpack/XML/实际显示槽。
- 剧情理解、人物关系、术语表只能辅助，不能覆盖 JP 原意。

## 1. 建立 JP baseline

必须确认：

- XML message 调用顺序
- egpack 文本槽
- speaker 表
- ruby 表
- id 是否重复
- XML 调用是否重复
- 被调用但 JP 是否为空
- 未调用文本是否存在
- speaker/ruby 是否缺失
- JP 是否混入英文整句

输出建议：

```text
chapters/<CHAPTER>/baseline/
```

## 2. 提取专有名词候选

至少覆盖：

- speaker 表名字
- 片假名人名
- 呼号 / 小队名
- 舰名 / 代号
- 军事组织 / 作战名
- 地名 / 势力名
- 英文缩写
- ruby 原词和读法
- 高频名词
- 只出现一次但像专名的词

用户确认后写入 main 共通术语表。

## 3. 建剧情/人物辅助理解包

在翻译前总结：

- 剧情时间线
- 主要人物表
- 人物关系
- 角色口吻
- 军衔/称呼规则
- 战斗通讯、独白、政治说明等语境注意点

注意：辅助理解包不能替代 JP 原文。

## 4. 初译

推荐：

- deepseek-v4-flash
- thinking disabled
- 输入只用 JP
- 输入前去掉/屏蔽 `\w`
- 输出 JSON：`{"cn":"..."}`
- 最终中文不保留 `\w`

按 300-500 行一批处理。

## 5. Codex 逐句校对

每句校对时带：

- 当前 id
- file / scene
- speaker
- JP 原文
- 初译中文
- 前 2-3 句 JP/CN
- 后 2-3 句 JP
- 命中术语
- 人物/剧情辅助提示

## 6. 批次审计

每批检查：

- 空中文
- 重复中文异常
- `jp:` 前缀残留
- JSON / placeholder 垃圾
- 英文整句残留
- 术语不一致
- speaker 名误进正文
- 控制符损坏
- Text ID Not Found 风险
- 繁体字 / 日文汉字残留

通过后提交一次 commit。

## 7. 总审计

全量完成后检查：

- 行数是否等于 JP baseline
- 调用顺序是否一致
- 空文本是否为 0
- Text ID Not Found 是否为 0
- 异常重复列表
- 术语表命中一致性
- speaker / ruby 问题
- 英文整句残留
- 繁体字 / 日文汉字残留
- 控制符检查

## 8. 打测试包

测试包必须：

- 带 README
- 带字体 payload
- install 前备份
- 不修改 Steam 原始文件
- 不修改存档
- zip 自检通过
- 可卸载或可还原备份

测试包不提交进 Git，只作为 Release 附件或网盘文件。

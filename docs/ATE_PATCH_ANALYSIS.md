# ATE 补丁思路总结

本文件记录从已有 ATE 汉化补丁中总结出的可复用方法。

## 核心启发

AGES 引擎文本可以通过解包、替换文本槽、重新打包的方式实现汉化补丁。关键不是直接修改 exe，而是构造游戏会读取的本地缓存/补丁 payload。

## 可复用原则

- 优先确认游戏实际读取哪个 egpack/XML/text 槽。
- 以 JP 源文本和调用顺序建立 baseline。
- speaker 表必须单独检查。
- ruby/注音/特殊显示文本不能漏。
- 字体必须一起处理，否则简中可能显示方块。
- 安装脚本应写入本地缓存并备份，避免修改 Steam 原始文件。

## 本项目额外防线

本项目早期在 TDA01-03 中遇到过这些问题：

- 英文槽混入导致台词错位。
- 旧中文/英文兜底导致重复台词。
- JP 空槽被粗暴清空，造成游戏日志显示空文本。
- speaker 表被英文名污染。
- 旧替换脚本把术语改坏。

因此后续流程固定为：

- 不使用英文槽作为翻译来源。
- 不使用旧中文兜底。
- 不使用模糊匹配回写。
- JP 空槽必须对照游戏实际显示槽。
- 重建前后都要做显示槽审计。
